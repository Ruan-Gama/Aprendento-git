git add nome-do-arquivo — Prepara o arquivo para o commit (adiciona ao "stage" ou "área de preparação")
git commit -m "mensagem do commit" — Registra as todas as alterações preparadas com uma mensagem descritiva (faltava espaço entre -m e as aspas)
touch nome-do-arquivo — Cria um arquivo vazio (esse é um comando do Linux/Mac, não do Git em si)
git log — Exibe o histórico de commits do repositório
git config — Configura opções do Git
git config --global | --local | --system user.name "Nome do Usuário" — Define o nome do usuário (a opção correta é --system, não "server"; e | deixa mais claro que são opções excludentes)
git config --global | --local | --system user.email email@exemplo.com — Define o e-mail do usuário
git status — Mostra o estado atual do repositório: quais arquivos foram modificados, estão no stage ou ainda não são rastreados pelo Git
.gitignore (é um arquivo, que dentro, você colocar o nome dos arquivos que você não quer salvar, como arquivos com informações sensiveis
.gitkeep(o github não reconhece diretorios vazios, e não salva, então é comum usar um arquivo .gitkeep, apenas para o git reconhecer e salvar uma pasta vazia
git add . (prepara todos os arquivos)
git restore nome-do-arquivo (se não tiver salvo, volta para o arquivo para como estava no ultimo commit
git --amend "mensagem-do-commit" (adiciona uma mensagem ao ultimo commit
git --amend (abre o editor para editar a mensagem do ultimo commit)
git reset --soft hash-do-commit (volta para o commit desejado, mas não apaga os arquivos do diretorio, apenas retorna os arquivos que ainda não existiam, para o estado de preparo do git add)
git reset --mixed hash-do-commit (volta para o commit desejado, mas não apaga os arquivos do diretorio, tira eles inclusive do estado de preparar, sendo necessario usar o git add denovo)
git reset --hard hash-do-commit (volta para o commit desejado completamente, apaga todos os arquivos que não existiam previamente)
git reflog (mostra o log, incluindo os resets eu acho)
git reset nome-do-arquivo (remove o arquivo da fase de preparação)
git restore --staged nome-do-arquivo (remove o arquivo da fase de preparação)
git remote add origin url-repositorio-online (conecta um repositorio do github com o repositorio local)
git branch -m nome-da-branch (cria uma branch, sempre crie uma main ao criar um repositorio
git push (envia as alterações do repositorio local para o remoto)
git push -u nome-da-branch-upstring nome-da-branch (seta uppstring dos caralho a 4, sei la mano n entendi que porra e essa explica direito)
git pull (envia as alterações do remoto para o local
